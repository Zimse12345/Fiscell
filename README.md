# Fiscell

Fiscell v1.1.21 是一个纯静态的本地记账网页项目，不需要后端和数据库。数据保存在当前浏览器的 `localStorage` 中，适合个人离线记账、导出备份、跨浏览器导入合并。

## 使用方式

直接用浏览器打开：

```text
E:\Code\Fiscell\index.html
```

部署到 GitHub Pages 后，也可以直接访问仓库发布地址。入口文件为 `index.html`。

项目没有构建步骤。当前机器没有检测到 Node/npm，所以应用保持为纯 HTML/CSS/JavaScript。

## 主要功能

- 添加支出、收入、理财记录
- 添加完成后保留上一次的分类、理财项目和具体标的预设
- 支出默认分类：餐饮、交通、购物、学习、娱乐、医疗、生活、其他
- 收入默认分类：工资、其他
- 理财默认操作：买入、做空，收益手动填写
- 理财操作金额视为资产内部转换，不改变总资产；理财收益会计入总资产
- 理财项目默认包括：基金、股票、期货、外汇、数字货币、其他
- 理财支持二级分类和具体标的，例如基金名称、证券代码、债券品种或产品名称
- 分类、理财项目和具体标的都可以直接手动输入，之后会出现在候选项里
- 支持对当前筛选条目执行本地智能归类，按备注和标签中的高敏感词置信分批量修正基础分类
- 支出和收入支持预结算标记：预收入、预支出
- 点击预收入或预支出卡片查看具体条目
- 点击总资产卡片恢复默认明细视图
- 在明细表中支持输入本次结算金额，也可以一键填入剩余金额；部分结算只更新当前条目，全部结算后会转成普通收入/支出明细
- 总资产、预收入、预支出、预期资产、灵活资产汇总
- 支持隐藏资产数字，隐藏后资产卡片、结构图和资产趋势图中的金额以星号显示
- 预期资产 = 总资产 + 预收入 - 预支出
- 灵活资产 = 总资产 - 未平仓理财资产
- 支持锚定资产，按总资产校准，存在偏差时自动生成其他收入或其他支出误差项
- 支持白色/黑色主题，点击太阳/月亮按钮切换
- 收入结构和支出结构可视化
- 点击收入/支出结构详情中的分类，可自动展开更多筛选并筛选该分类
- 资产变化折线图支持近30天、近90天、近一年和全部范围，带淡化面积背景，可悬停查看资产，也可点击或拖选时间区间筛选明细，并按跨度自动调整横轴刻度
- 更多筛选支持按时间、金额区间和分类筛选
- 账目明细表头支持按金额或时间升降序排序，默认按时间降序显示，清空筛选不会重置排序
- 批量选择后支持跨分页全选、清除、批量删除、导出和设置收入/支出分类
- 理财收益计入收入结构，理财亏损计入支出结构，分类为“理财”
- JSON 导出备份
- CSV 导出到表格软件
- JSON/CSV 导入前预检查数量、类型、时间范围和重复 ID，确认后覆盖或合并
- 导入说明中支持查看转换流程并一键复制账单转换提示词

## 数据合并规则

每条记录都有稳定的 `id`。导入时：

- 本地没有同 `id` 的记录：新增
- 本地已有同 `id` 的记录：保留 `updatedAt` 更新的一条
- 确认导入前会预览条目数量、收入/支出/理财数量、时间范围和重复 ID
- 确认导入后会把当前账本保存到 `local-ledger.last-import-backup.v4`

CSV 或旧账单没有 `id` 时，会用时间、类型、金额、分类、理财项目、具体标的、备注和行号生成一个稳定导入 ID。重复导入同一份文件时，会尽量合并为同一批记录。

## 旧账单导入规则

批量导入先前账单时，如果缺少类型或详情：

- 默认视作收入
- 分类默认为其他
- 金额为负数时推断为支出
- 有支出列时推断为支出
- 有收入列时推断为收入

## AI 转换提示词

如果你有银行、微信、支付宝或其他格式账单，可以把文件内容交给另一个 AI，并使用 [AI_IMPORT_PROMPT.md](./AI_IMPORT_PROMPT.md) 里的提示词转换成此项目可导入的 JSON。
